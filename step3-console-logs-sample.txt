==============================================

The above cycle repeats thousands of times until the browser crashes.

KEY OBSERVATIONS:
- finalSubmitAttempted toggles: false → true → false → true → ...
- attemptedSubmit stays true after the first toggle
- All 6 form fields re-render on every toggle
- useEffect fires on every cycle with "Clearing errors for Step 3"
- clearErrors() runs on every cycle
- No errors are ever present (errors object is always {})

PERFORMANCE IMPACT:
- Each cycle renders 6 form fields
- The cycle repeats approximately 1000+ times per second
- Total renders: 6000+ component renders per second
- Browser freezes within 2-3 seconds
- Page becomes completely unresponsive

ROOT CAUSE HYPOTHESIS:
The useEffect that clears errors is causing the infinite loop:
1. useEffect runs because of a dependency change
2. clearErrors() is called
3. Calling clearErrors() updates state
4. State update triggers re-render
5. Re-render causes useEffect to run again (dependency changed)
6. Loop repeats infinitely

SOLUTION:
Fix the useEffect dependencies or add a guard condition to prevent
the infinite loop. The clearErrors() call should only run once when
Step 3 first mounts, not on every render.
